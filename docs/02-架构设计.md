### 2.2 核心表结构

#### sys_user (用户表)
```sql
CREATE TABLE sys_user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password VARCHAR(255) NOT NULL COMMENT '密码 (BCrypt)',
    email VARCHAR(100) COMMENT '邮箱',
    phone VARCHAR(20) COMMENT '手机号 (加密存储)',
    nickname VARCHAR(50) COMMENT '昵称',
    avatar VARCHAR(255) COMMENT '头像 URL',
    gender TINYINT DEFAULT 0 COMMENT '性别 0-未知 1-男 2-女',
    org_id BIGINT COMMENT '组织 ID',
    dept_id BIGINT COMMENT '部门 ID',
    post_ids VARCHAR(500) COMMENT '岗位 ID 列表，逗号分隔',
    status TINYINT DEFAULT 1 COMMENT '状态 0-禁用 1-正常',
    is_admin TINYINT DEFAULT 0 COMMENT '是否超级管理员',
    login_ip VARCHAR(50) COMMENT '最后登录 IP',
    login_date DATETIME COMMENT '最后登录时间',
    pwd_update_time DATETIME COMMENT '密码最后修改时间',
    deleted TINYINT DEFAULT 0 COMMENT '删除标记',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_username (username),
    INDEX idx_org_dept (org_id, dept_id),
    INDEX idx_status (status)
);
```

#### sys_role (角色表)
```sql
CREATE TABLE sys_role (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    role_code VARCHAR(50) NOT NULL UNIQUE COMMENT '角色编码',
    role_name VARCHAR(100) NOT NULL COMMENT '角色名称',
    data_scope TINYINT DEFAULT 4 COMMENT '数据范围 1-全部 2-本部门及子部门 3-本部门 4-本人 5-自定义',
    custom_dept_ids VARCHAR(500) COMMENT '自定义部门 ID 列表',
    sort INT DEFAULT 0 COMMENT '排序',
    status TINYINT DEFAULT 1 COMMENT '状态',
    deleted TINYINT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_code (role_code)
);
```

#### sys_menu (菜单权限表)
```sql
CREATE TABLE sys_menu (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    menu_code VARCHAR(50) NOT NULL UNIQUE COMMENT '菜单编码',
    menu_name VARCHAR(100) NOT NULL COMMENT '菜单名称',
    parent_id BIGINT DEFAULT 0 COMMENT '父菜单 ID',
    path VARCHAR(200) COMMENT '路由路径',
    component VARCHAR(200) COMMENT '组件路径',
    type TINYINT NOT NULL COMMENT '类型 1-目录 2-菜单 3-按钮',
    permission VARCHAR(100) COMMENT '权限标识',
    icon VARCHAR(100) COMMENT '图标',
    sort INT DEFAULT 0 COMMENT '排序',
    visible TINYINT DEFAULT 1 COMMENT '是否可见',
    status TINYINT DEFAULT 1 COMMENT '状态',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_parent (parent_id),
    INDEX idx_type (type)
);
```

#### sys_data_scope_rule (数据权限规则表)
```sql
CREATE TABLE sys_data_scope_rule (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    role_id BIGINT NOT NULL COMMENT '角色 ID',
    table_name VARCHAR(100) NOT NULL COMMENT '表名',
    column_name VARCHAR(100) NOT NULL COMMENT '列名 (如：create_by, dept_id)',
    scope_type TINYINT NOT NULL COMMENT '范围类型',
    priority INT DEFAULT 0 COMMENT '优先级',
    UNIQUE KEY uk_role_table (role_id, table_name),
    INDEX idx_role (role_id)
);
```

#### sys_app (开放平台应用表)
```sql
CREATE TABLE sys_app (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    app_key VARCHAR(50) NOT NULL UNIQUE COMMENT '应用 Key',
    app_secret VARCHAR(255) NOT NULL COMMENT '应用密钥',
    app_name VARCHAR(100) NOT NULL COMMENT '应用名称',
    owner_id BIGINT COMMENT '负责人 ID',
    callback_url VARCHAR(255) COMMENT '回调 URL',
    allowed_ips TEXT COMMENT 'IP 白名单，JSON 数组',
    rate_limit INT DEFAULT 1000 COMMENT 'QPS 限制',
    status TINYINT DEFAULT 1 COMMENT '状态',
    expires_at DATETIME COMMENT '过期时间',
    last_used_at DATETIME COMMENT '最后使用时间',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_appkey (app_key)
);
```

### 4.1 JWT Token 结构

```json

  "exp": 1714298400,
  "jti": "uuid-xxx-xxx"

```

### 4.2 缓存策略

| 数据 | 缓存方式 | 过期策略 |
|------|----------|----------|
| Token 信息 | Redis String | 与 Token 有效期一致 |
| 用户信息 | Redis Hash | 30 分钟 + 滑动过期 |
| 权限信息 | Redis Set | 30 分钟 + 事件失效 |
| 菜单树 | Redis String(JSON) | 1 小时 + 事件失效 |
| 应用密钥 | Redis String | 永不过期 |

## 5. 部署架构

### 5.1 Docker Compose 部署

```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_PASSWORD}
      
  redis:
    image: redis:7-alpine
    
  account-service:
    build: ./account-service
    deploy:
      replicas: 3
    environment:
      SPRING_PROFILES_ACTIVE: prod
      
  nginx:
    image: nginx:alpine
    ports:
      - "443:443"
```

### 5.2 生产环境建议

- **应用层**: 至少 2 节点，K8s HPA 自动扩缩容
- **数据库**: 主从复制，读写分离
- **缓存**: Redis Cluster 3 主 3 从
- **监控**: Prometheus + Grafana + AlertManager
